В текущей версии использованы демонстрационные данные для отзывов (`data/reviews_raw.csv`), так как API-ключи для 2GIS/Google/Twitter отсутствуют.

## Ответ №1 (анализ тональности отзывов)

### Цель
Собрать отзывы конкурентов (минимум 3 компании), классифицировать их по тональности (negative/neutral/positive), оценить качество классификации и динамику изменений во времени.

### Что сделано

1. Подготовлен скрипт сбора отзывов через API:
   - `reviews_api.py`
   - поддержаны источники: 2GIS, Google Places, Twitter/X.

2. Подготовлен скрипт анализа тональности:
   - `sentiment_analysis.py`
   - модель: `blanchefort/rubert-base-cased-sentiment` (3 класса: negative/neutral/positive).

3. Выполнена оценка качества:
   - weak supervision через рейтинг (1-2 = negative, 3 = neutral, 4-5 = positive),
   - сформирован отчет метрик: `plots_sentiment/quality_report.txt`.

4. Проанализирована динамика отзывов:
   - доля тональностей по месяцам,
   - средний рейтинг по месяцам,
   - сравнение тональности по конкурентам.

### Ссылки на код и данные

- Код:
  - `reviews_api.py`
  - `sentiment_analysis.py`
- Данные:
  - `data/reviews_raw.csv`
  - `data/reviews_scored.csv`
- Результаты:
  - `plots_sentiment/sentiment_share_over_time.png`
  - `plots_sentiment/avg_rating_over_time.png`
  - `plots_sentiment/sentiment_by_competitor.png`
  - `plots_sentiment/quality_report.txt`

### Краткий вывод
Технически задача классификации тональности решена программно, включая оценку качества и временную динамику. Для финальной бизнес-интерпретации требуется заменить демонстрационные отзывы на реальные API-выгрузки конкурентов.

## Ответ №2 (Google Trends 2005–2025)

### Цель
Получить и проанализировать временной ряд интереса к поисковому термину, исследовать стационарность, построить ACF и прогноз с горизонтом 10.

### Что сделано

1. Реализован скрипт:
   - `trends_analysis.py`
   - период: 2005-01-01 — 2025-12-31,
   - термин: `смартфон`,
   - регион: `KZ`.

2. Построены визуализации:
   - временной ряд,
   - ACF (автокорреляционная функция),
   - прогноз с доверительным интервалом.

3. Проверена стационарность:
   - ADF-тест сохранен в `plots_trends/stationarity_adf.txt`.

4. Прогноз:
   - модель SARIMAX baseline,
   - горизонт: 10 шагов.

### Ссылки на код и данные

- Код:
  - `trends_analysis.py`
- Данные:
  - `data/trends_смартфон_KZ.csv`
- Результаты:
  - `plots_trends/trends_timeseries.png`
  - `plots_trends/trends_acf.png`
  - `plots_trends/trends_forecast.png`
  - `plots_trends/stationarity_adf.txt`

### Краткий вывод
Интерес к термину имеет выраженную динамику во времени и автокорреляционную структуру. Прогноз на 10 шагов показывает ожидаемое продолжение тренда в рамках доверительного интервала модели.

## Ответ №3 (дашборд и интерпретация)

### Что подготовлено для BI-дашборда (Power BI / Tableau / Qlik)

Основной датасет для дашборда:
- `data/reviews_scored.csv`

Рекомендуемые страницы дашборда:

1. **Общий обзор тональности**
   - KPI: доля negative/neutral/positive,
   - фильтры: competitor, source, период.

2. **Динамика во времени**
   - line chart: доля negative/neutral/positive по месяцам,
   - line chart: средний rating по месяцам.

3. **Сравнение конкурентов**
   - stacked bar: распределение тональности по компаниям,
   - таблица: число отзывов, доля негатива, средний рейтинг.

4. **Слабые места конкурентов (negative отзывы)**
   - фильтр `sentiment = negative`,
   - выделение частых проблемных тем (сервис, доставка, гарантия, цена, качество).

### Оценка будущих тенденций

- Рост доли негативных отзывов по компании может сигнализировать об ухудшении клиентского опыта.
- Снижение средней оценки при росте объема отзывов — риск репутационной просадки.
- Компании с устойчивой долей позитивных отзывов и низкой волатильностью выглядят наиболее стабильными.

### Описание дашбордов

Дашборды собираются на основе:
- `data/reviews_scored.csv`
- изображений из `plots_sentiment/` и `plots_trends/` как визуальных подтверждений аналитики.
